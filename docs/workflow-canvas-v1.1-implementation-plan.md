# 工作流画布 V1.1 开发任务拆解清单

版本：`v1.1`  
日期：`2026-03-05`  
关联需求：`docs/workflow-canvas-prd-v1.1.md`

## 1. 交付目标

1. 实现单配方/多配方统一执行模型。
2. 支持广播/一一对应/全相乘三种组合模式。
3. 支持模板复用时动态/固定槽位行为。
4. 支持失败暂停、已启动任务继续、单任务重试。
5. 结果页按素材分组展示与追溯。

## 2. 里程碑（建议 4 个迭代）

1. `M1`：数据模型 + API 基线 + 执行引擎最小可跑
2. `M2`：画布配置 UI（组合模式/配方集/槽位）+ 运行前校验
3. `M3`：可观测性 + 失败恢复 + 单任务重试
4. `M4`：模板复用闭环 + 结果分组页 + 回归测试

## 3. 后端任务清单

### 3.1 数据结构与持久化

1. 新增 `WorkflowRunSnapshot`：
1. `run_id`
2. `graph_version`
3. `assets_input`
4. `prompts_input`
5. `recipes`
6. `combination_mode`
7. `expanded_tasks`
8. `status`
9. `created_at/updated_at`
2. 新增 `ExpandedTask`：
1. `task_id`
2. `source_asset_id`
3. `prompt_id`
4. `recipe_id`
5. `variant_index`
6. `status`
7. `error`
8. `output_asset_ids`
9. `timing`
3. 新增模板槽位定义：
1. `slot_name`
2. `slot_type`（dynamic/fixed）
3. `required`
4. `bound_asset_id`（fixed 时必填）

### 3.2 运行编排

1. 实现组合展开器 `expand_tasks()`：
1. 输入：素材、提示词、配方、组合模式、每图张数
2. 输出：任务列表（最多 50）
2. 实现执行调度器 `run_tasks()`：
1. 并发数控制
2. 失败暂停（停止派发新任务）
3. 已启动任务完成后写回结果
4. 支持单任务重试 `retry_task(task_id)`
3. 保留已有并发写入锁机制并扩展到 run snapshot 写入。

### 3.3 API 接口（新增/扩展）

1. `POST /api/v1/workflow/runs/preview`
1. 输入配置，返回任务总数与样例任务预览
2. 用于运行前校验和超限提示
2. `POST /api/v1/workflow/runs`
1. 创建并启动一次运行
3. `GET /api/v1/workflow/runs/{run_id}`
1. 获取运行状态、步骤、任务明细、产出映射
4. `POST /api/v1/workflow/runs/{run_id}/pause`
5. `POST /api/v1/workflow/runs/{run_id}/resume`
6. `POST /api/v1/workflow/runs/{run_id}/tasks/{task_id}/retry`
7. 模板接口扩展：
1. 保存/读取槽位定义与配方集

## 5. 校验与业务规则实现

1. 一一对应模式：素材与提示词数量必须相同。
2. 全相乘模式：允许数量不等。
3. 动态槽位：运行时必须绑定新输入素材。
4. 固定槽位：运行时保持固定绑定。
5. 任务上限：`50`，预估阶段拦截。
6. 重试粒度：仅单任务，不影响成功任务。

## 6. 测试清单

### 6.1 单元测试

1. `expand_tasks()` 三种模式展开正确性。
2. 任务上限校验（49/50/51 边界）。
3. 槽位解析逻辑（动态/固定）。
4. 失败暂停状态机。

### 6.2 集成测试

1. 3图 + 1词 + 每图2张 -> 6任务。
2. 5图 + 5词 + 一一对应 -> 5任务。
3. 5图 + 3词 + 全相乘 -> 15任务。
4. 运行中失败 -> 暂停 -> 单任务重试 -> 恢复完成。
5. 模板复用时动态槽位自动替换，固定槽位不替换。

### 6.3 回归测试

1. 现有开始/结束/输入/合并节点约束不回归。
2. 现有文本、生图节点 API 调用不回归。
3. 并发下会话写入与结果持久化无冲突。

## 7. 上线前检查

1. 日志包含 `run_id/task_id/node_id` 三元追踪。
2. 失败消息可直接定位上游错误原因。
3. 前端运行状态与后端状态一致。
4. 模板导入导出兼容旧版本字段（缺省值回填）。

## 8. 风险与应对

1. 风险：全相乘导致任务爆炸。  
应对：运行前预估 + 50上限硬拦截。

2. 风险：并发下部分任务随机网络失败。  
应对：保留瞬时错误重试 + 失败暂停 + 单任务重试。

3. 风险：模板复用时槽位绑定错配。  
应对：必填槽位校验 + 启动前摘要确认弹窗。

## 9. 建议实施顺序（可直接开工）

1. 后端先完成 `preview + run + retry` 三接口和任务展开器。
2. 前端接入启动前预估与组合模式，先打通最小链路。
3. 再补配方编辑与槽位绑定 UI。
4. 最后补结果分组页和模板复用细节。
