# 工作流画布产品需求文档（PRD）

版本：`v1.1`  
状态：`Draft / 可进入开发拆解`  
日期：`2026-03-04`  
适用范围：`studio_aligned 新增工作流画布能力（与 xhs-studio-unified 打通）`

## 1. 背景

当前主流生图操作以“对话式单轮调用”为主，难以覆盖真实工作中的复杂链路。实际生产需要的是可复用、可批量、可观测、可人工接管的流程化能力，而不是重复手工点击。

典型痛点：

1. 输入组合复杂：单图、批量图、上游结果回流。
2. 提示词组合复杂：单提示词、多提示词、模板化提示词。
3. 产出规模复杂：一一对应、全相乘、每图多候选。
4. 失败处理复杂：并发下随机失败，缺少暂停与细粒度重试。
5. 复用复杂：流程可保存，但替换新素材继续跑不够顺滑。

## 2. 目标与原则

### 2.1 产品目标

1. 用一套画布流程同时支持“单配方”和“多配方”。
2. 支持批量输入、批量执行、批量产出，减少重复劳动。
3. 支持模板复用，用户替换输入即可继续同流程生产。
4. 提供运行可观测性与人工接管，保证可控性。
5. 与现有程序 API 和素材体系打通，避免重复系统。

### 2.2 设计原则

1. 先简单后高级：默认配置最少，进阶能力折叠到高级设置。
2. 一套机制覆盖多场景：单配方与多配方不分两套引擎。
3. 可解释：每个结果可追溯到输入、配方、参数、步骤。
4. 失败可恢复：暂停后保留已产出，支持单任务重试。

## 3. 非目标（V1 不做）

1. 多引擎自动路由（按成本/速度自动切模型）不在 V1。
2. seed 固定复现能力不在 V1（当前 API 无 seed 概念）。
3. 超大规模批任务调度不在 V1（单次上限先定 50 任务）。

## 4. 核心概念模型

1. 素材集：本次运行输入的图片集合。
2. 提示词集：本次运行使用的提示词集合。
3. 配方集：规则集合，每条配方定义“如何生成”。
4. 组合模式：素材与提示词如何展开为任务。
5. 任务单元：一次模型调用。
6. 运行快照：一次运行的完整可追溯记录。

统一计算公式：

`任务总数 = 素材数 × 提示词数 × 配方数 × 每图产出张数`

## 5. 组合模式定义

V1 支持 3 种组合模式：

1. 广播模式  
定义：一边固定，另一边批量。  
示例：5 图 + 1 词 -> 5 任务；1 图 + 5 词 -> 5 任务。

2. 一一对应模式  
定义：按索引配对。  
示例：图1-词1，图2-词2。  
约束：数量必须相等，否则禁止运行并给出提示。

3. 全相乘模式  
定义：笛卡尔积。  
示例：5 图 × 3 词 -> 15 任务。  
说明：5 图 3 词在此模式下可正常运行。

## 6. 配方维度定义

“配方”是规则，不是画布上的并行分支。  
单配方与多配方只在 `recipes[]` 的条目数量不同，引擎一致。

每条配方最小字段：

1. `recipe_id`
2. `name`
3. `prompt_template`
4. `model_type`（V1 先聚焦生图）
5. `model`
6. `aspect_ratio`
7. `quality`
8. `reference_binding`（素材槽位映射）
9. `variants_override`（可选，不填则继承全局每图产出张数）

## 7. 槽位与引用规则（V1.1 关键增补）

### 7.1 槽位类型

1. 动态槽位  
模板仅保存槽位定义，不保存具体素材 ID。  
调用模板时，自动用“本次上传素材”替换。

2. 固定槽位  
模板保存具体素材引用。  
调用模板时，不自动替换。

默认建议：

1. 批处理相关槽位默认动态槽位。
2. 品牌长期基准素材默认固定槽位。

### 7.2 引用输入体验

1. 素材输入区支持 `@` 快速检索并插入素材。
2. 提示词编辑区支持 `@` 引用全局素材与上游结果。
3. 前端允许自由插入，后端执行前统一解析为结构化槽位绑定。

### 7.3 运行前校验

1. 必填动态槽位未绑定，禁止启动。
2. 一一对应模式数量不一致，禁止启动。
3. 任务总数超过上限 50，禁止启动。

## 8. 节点体系与连线规则

### 8.1 节点类型

1. 开始节点
2. 输入节点（执行模型）
3. 审核节点（人工接管）
4. 显示节点
5. 转换节点（可选，文本/结构转换）
6. 合并节点（文字拼接/素材聚合）
7. 存储节点
8. 结束节点

### 8.2 连线约束

1. 开始节点：允许多个；无入边；仅一个出边。
2. 结束节点：全流程仅一个；无出边；仅一个入边。
3. 输入节点：可接上游也可接下游；入边最多 2 条。
4. 合并节点：允许多入单出。
5. 多分支汇总后进结束节点时，必须先进入合并节点。

## 9. 执行引擎规则

### 9.1 调度与并发

1. 默认开启并发。
2. 并发数可配置。
3. 单次任务上限：50。
4. 运行前显示任务预估与并发计划。

### 9.2 失败与恢复

1. 任一任务失败后，状态转 `paused`。
2. 停止派发新任务。
3. 已经开始的任务继续执行并保存产出。
4. 恢复时按单任务粒度重试，不重跑已成功任务。

### 9.3 上游结果回流

1. 任意节点产出可作为下游输入。
2. “多素材 + 单提示词”中的素材来源可包含“上一步单图流程结果”。
3. 该能力属于主流程能力，不视为额外需求。

## 10. 模板系统

### 10.1 模板能力

1. 保存模板
2. 编辑模板
3. 删除模板
4. 从模板启动运行

### 10.2 模板保存内容

1. 图结构（节点/边）
2. 配方集定义
3. 槽位定义（动态/固定）
4. 组合模式
5. 参数默认值（并发、每图产出张数等）

### 10.3 模板复用规则

1. 动态槽位在每次运行自动映射到新输入素材。
2. 固定槽位保持原引用不变。
3. 允许“固定提示词库 + 新素材输入”重复执行同模板。

## 11. 可观测性与日志

1. 每步骤展示状态：`pending/running/completed/failed/paused`。
2. 展示耗时（秒级）。
3. 前端轮询周期：2s。
4. 错误需可见具体原因（如网络失败、参数校验失败、上游返回失败）。
5. 支持后端日志快速定位（工作流、任务、节点维度）。

## 12. 结果映射与存储

### 12.1 结果映射主键

每个结果至少携带：

1. `source_asset_id`
2. `prompt_id`
3. `recipe_id`
4. `variant_index`

### 12.2 结果页展示

1. 默认按素材分组查看结果（用户已确认）。
2. 可在组内查看不同配方、提示词、候选版本。
3. 支持对失败任务进行单任务重试。

### 12.3 统一存储页

1. 流程产出
2. 模板库
3. 素材库

## 13. 与现有系统打通要求

1. 新建独立 Web 应用承载画布能力。
2. 与 `@xhs-studio-unified` 打通统一 API 入口。
3. 模型调用在节点内按“文字/生图”类型选择并执行。
4. 新程序可引用旧系统素材与产出。
5. 不改变旧程序当前能力，通过切换入口进入画布模式。

## 14. 场景覆盖清单（V1 必须支持）

1. 单素材 + 单提示词 -> 单结果。
2. 多素材 + 单提示词 -> 批量一致处理。
3. 单素材 + 多提示词 -> 多风格探索。
4. 多素材 + 多提示词一一对应 -> 批量成对处理。
5. 多素材 + 多提示词全相乘 -> 矩阵探索。
6. 上游结果作为下游素材。
7. 模板复用时替换新素材继续跑。

## 15. 示例

### 15.1 你提出的核心示例

输入：

1. 装饰图 3 张（批量）
2. 沙发素材 1 张（全局参考）
3. 固定提示词 1 条
4. 每图产出张数 = 2

执行：

1. 组合模式：广播（3图 x 1词）
2. 任务数：`3 × 1 × 1 × 2 = 6`
3. 并发执行，失败按暂停策略处理

输出：

1. 总计 6 张结果
2. 结果按原装饰图分组展示

### 15.2 5 图 + 5 词 + 一一对应

任务数：`5 × 1 × 1 × 1 = 5`（按配对关系）

### 15.3 5 图 + 3 词 + 全相乘

任务数：`5 × 3 × 1 × 1 = 15`

## 16. 运行限制与保护

1. 任务上限：50。
2. 启动前必须给出任务总数预估。
3. 组合展开后若超上限，必须阻断并提示。
4. 模板保存前进行完整性校验（必填槽位、非法连线、结束节点唯一性）。

## 17. 验收标准（UAT）

1. 用户可通过画布完成“开始 -> 输入/批处理 -> 合并 -> 结束”的可运行流程。
2. 在 3 图 + 1 词 + 每图 2 张场景下，一次启动并得到 6 张结果。
3. 发生失败时流程暂停，已启动任务继续完成并落库，支持单任务重试。
4. 模板保存后，替换素材可复跑，动态槽位自动更新，固定槽位不变。
5. 结果页按素材分组准确展示，并可追溯每个结果对应的配方与提示词。

## 18. 里程碑建议

1. M1：组合模式 + 批处理执行 + 失败暂停。
2. M2：模板槽位（动态/固定）+ 模板复用。
3. M3：结果映射完善 + 存储页分组与重试。
4. M4：性能与稳定性优化（并发、日志、错误分类）。
