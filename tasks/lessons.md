# Lessons

- 当用户提出“默认打开状态”和“不保存也要恢复”的体验诉求时，优先实现本地草稿持久化（自动保存 + 自动恢复），不要仅依赖模板保存。
- 连线规则要按节点类型分层定义（如 `input` 可限额多入线），避免“全局单入线”影响真实业务场景。
- 当工作流从串行改为并发后，必须同步审计后端持久化写入路径（尤其是固定 `.tmp` 原子写模式），否则会出现并发分支随机 `Failed to fetch`。
- 并发工作流中，模型上游出现 `EOF/_ssl` 这类瞬时网络抖动是高频真实场景；应在“未产出 token/结果”时自动重试，并把 4xx 业务错误与可重试网络错误分开处理。
- 用户对“批量模式”最容易困惑的是映射关系；界面必须优先给出步骤化引导和实时配对预演，不能只给参数输入框。
- 当功能依赖提示词工程（如分步生图）时，模板必须配置化并暴露到设置层，避免硬编码导致用户无法按业务场景迭代。
- 只靠文字约束“保持结构不变”不够，实际生图阶段要默认携带原参考图一起请求，才能显著降低结构漂移。
